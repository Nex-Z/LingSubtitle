# 灵幕 (LingSubtitle)

一款基于系统音频的实时语音字幕桌面应用，支持捕获浏览器、视频软件、会议软件等播放的音频，基于阿里云 Gummy 实时输出原文与译文，并提供可悬浮的双语字幕展示。

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Tauri 2 |
| 前端 | React + TypeScript + Vite |
| 后端 | Rust |
| 平台 | Windows（优先）→ Mac（后续） |

---

## 功能需求

### 1. 音频捕获

#### 1.1 系统内录（Loopback）
- 捕获系统所有音频输出（系统混音/回环录制）
- 适用场景：浏览器视频、在线会议、音乐播放等所有系统音频
- Windows 上使用 WASAPI Loopback 模式实现

#### 1.2 指定软件捕获
- 仅捕获用户指定的某个应用程序的音频输出
- 不受其他应用程序音频干扰
- 用户可从运行中的应用列表中选择目标进程
- Windows 上使用 WASAPI 进程级音频捕获（Windows 10 1903+）

#### 1.3 切换机制
- 用户可在「系统内录」和「指定软件」两种模式之间自由切换
- 切换时无需重启应用

---

### 2. 语音识别（Speech-to-Text）

#### 2.1 在线识别 — 阿里云 Gummy 实时任务（当前实现）
- 已接入 **gummy-realtime-v1**
- 基于 WebSocket 双工任务流，启动录制时发送 `run-task`，停止录制时发送 `finish-task`
- 识别与翻译共用一条服务端实时任务流，不再走“ASR + 独立文本翻译”两段式
- 支持自动识别或手动指定源语言
- 音频格式：PCM 16-bit 16kHz 单声道
- 句子边界以服务端返回的 `sentence_id + sentence_end` 为准，前端不再本地软分句
- 支持 VAD 句尾静音阈值配置

#### 2.2 音频处理管线
- **立体声→单声道混音**：多声道音频按帧平均为单声道
- **重采样至 16kHz**：线性插值将系统采样率（通常 48kHz）降采样至 ASR 所需的 16kHz
- **低延迟刷新**：50ms 音频缓冲刷新间隔，零拷贝 buffer 切换

#### 2.3 多语言支持
- 支持多种语言的语音识别
- **支持自动语言检测**：翻译关闭时可直接使用自动识别
- 首页可快速切换识别语种

---

### 3. 自动翻译

#### 3.1 在线翻译 — Gummy 实时翻译（当前实现）
- 翻译与识别共用同一条 Gummy 实时任务流
- 翻译开启时才向服务端申请译文，便于控制成本
- 当前仅支持单一目标语言
- 语言组合严格按 Gummy 官方能力矩阵约束
- 翻译开启时不允许源语言使用自动识别
- 翻译开关和目标语言在首页控制 Dock 中即时切换

#### 3.2 翻译显示
- 翻译开启时：译文为主显示，原文为参考
- 翻译关闭时：仅显示原文
- 悬浮窗支持双语固定双行布局

---

### 4. 悬浮字幕窗口

#### 4.1 始终置顶
- 字幕窗口支持 **始终置顶（Always-on-Top）** 模式
- 可覆盖在其他应用窗口之上显示
- 自动定位屏幕下 1/3 处

#### 4.2 固定双行布局
- **上行**：主文字（有翻译时显示译文，否则显示原文）
- **下行**：原文参考（翻译开启时平滑展开，关闭时折叠）
- 深色毛玻璃背景，文字清晰可读
- 平滑 CSS 过渡动画，无布局跳动

#### 4.3 自定义设置
- **字号选择**：14 / 16 / 18 / 22 / 26px
- **背景透明度**：30%～100% 可调
- **显示原文**：可关闭原文参考行
- 设置自动保存至 localStorage

#### 4.4 窗口交互
- 顶部拖拽区域
- 悬浮显示设置和关闭按钮
- 可自由调整窗口大小

---

### 5. 自动保存

#### 5.1 保存格式
- 保存为带时间戳的纯文本文件（`.txt`）
- 格式示例：
  ```
  [2026-03-09 14:30:15] Hello, welcome to the meeting.
  [2026-03-09 14:30:15] 你好，欢迎参加会议。
  [2026-03-09 14:30:22] Let's start with today's agenda.
  [2026-03-09 14:30:22] 让我们从今天的议程开始。
  ```

#### 5.2 保存路径
- 默认保存路径：用户文档目录下的 `LingSubtitle` 文件夹
- 保存路径可在设置中自定义配置
- 文件名自动生成，格式：`字幕_YYYY-MM-DD_HH-mm-ss.txt`

#### 5.3 历史记录
- 提供历史记录列表页面，可查看所有已保存的字幕文件
- 支持按日期筛选和搜索
- 仅支持**查看**（只读浏览），不支持回放
- 支持打开文件所在目录

---

### 6. 设置与控制

#### 6.1 首页实时控制
- 捕获模式选择（系统内录 / 指定软件）
- 指定应用列表刷新与切换
- 识别语言快速切换
- 翻译开关与目标语言即时切换
- 服务状态检测与录制状态展示

#### 6.2 设置页
- Gummy 识别设置：
  - WebSocket 地址
  - API Key
  - 模型名称
  - 热词表 ID
  - 句尾静音阈值与 VAD 预设
  - 过滤语气词
- 保存设置：
  - 自动保存开关
  - 保存路径配置

#### 6.3 悬浮窗设置
- 字号、透明度、是否显示原文
- 设置保存在 localStorage

## 非功能性需求

### 性能要求
| 指标 | 目标 |
|------|------|
| 字幕延迟 | < 2 秒（从音频发出到字幕显示） |
| 长时间运行 | ≥ 8 小时无崩溃、无内存泄漏 |
| CPU 占用 | 离线识别时 < 30%（中等配置 PC） |
| 内存占用 | < 1 GB（含模型加载） |

### 稳定性要求
- 长时间运行（≥ 8 小时）不出现内存泄漏
- 音频流中断后可自动恢复
- 异常情况下已录制的字幕不丢失（定时写入文件）

### 兼容性
- Windows 10 1903+ / Windows 11
- Mac 支持后续迭代（Tauri 2 本身跨平台）

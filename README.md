# 灵幕 (LingSubtitle)

一款基于系统音频的实时语音字幕桌面应用，支持捕获浏览器、视频软件、会议软件等播放的音频，实时展示可悬浮的语音字幕。

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Tauri 2 |
| 前端 | React + TypeScript + Vite |
| 后端 | Rust |
| 平台 | Windows（优先）→ Mac（后续） |

---

## 功能需求

### 1. 音频捕获

#### 1.1 系统内录（Loopback）
- 捕获系统所有音频输出（系统混音/回环录制）
- 适用场景：浏览器视频、在线会议、音乐播放等所有系统音频
- Windows 上使用 WASAPI Loopback 模式实现

#### 1.2 指定软件捕获
- 仅捕获用户指定的某个应用程序的音频输出
- 不受其他应用程序音频干扰
- 用户可从运行中的应用列表中选择目标进程
- Windows 上使用 WASAPI 进程级音频捕获（Windows 10 1903+）

#### 1.3 切换机制
- 用户可在「系统内录」和「指定软件」两种模式之间自由切换
- 切换时无需重启应用

---

### 2. 语音识别（Speech-to-Text）

#### 2.1 离线识别
- 集成本地离线语音识别引擎
- **推荐方案**：Whisper.cpp（基于 OpenAI Whisper 模型的 C/C++ 实现）
  - 英文干净音频 WER（词错误率）约 2~5%
  - 支持 99+ 种语言及自动语言检测
  - 中小模型可在 CPU 上以接近实时的速度运行（推理延迟约 500ms）
  - 内存占用合理，适合长时间运行
- **备选方案**：Sherpa-ONNX / Vosk
  - Sherpa-ONNX：基于 ONNX Runtime，支持流式识别，跨平台部署
  - Vosk：超轻量级，适合低延迟边缘设备场景
- 用户可选择不同大小的模型（如 tiny / base / small / medium）以平衡精度和性能

#### 2.2 在线识别 — 大模型 ASR
- 支持接入大语言模型（LLM）的语音识别能力
- **最低要求：兼容 OpenAI API 格式**
  - 原生支持：OpenAI Whisper API
  - 兼容接入：任何提供 OpenAI 兼容 ASR API 的服务，包括但不限于：
    - 阿里通义（Qwen-Audio / Paraformer）
    - 讯飞星火
    - Azure OpenAI
    - 本地部署的兼容服务
- 用户可自定义配置：
  - API 地址（Base URL）
  - API Key
  - 模型名称
- 适用于对识别精度有更高要求，或需要特定语言优化的场景

#### 2.3 引擎切换
- 用户可在设置中自由选择「离线」或「大模型」识别模式
- 离线模式：无需联网，隐私保护好，零使用成本
- 大模型模式：需要网络，精度更高，可能产生费用

#### 2.4 多语言支持
- 支持多种语言的语音识别（中文、英文、日文、韩文等）
- **支持自动语言检测**：应用自动识别音频所使用的语言，用户无需手动选择

> **离线识别效果分析**：根据调研，Whisper.cpp 在使用 small 及以上模型时，对主流语言（英/中/日/韩/欧洲语言）的识别准确率已接近云端服务水平，WER 可控制在 5~10% 以内。对于字幕展示场景（非精确转录），该精度完全可用。离线模式的主要优势在于零延迟网络开销、完全隐私保护和零使用成本。

---

### 3. 自动翻译

#### 3.1 离线翻译
- 集成本地离线翻译模型
- **推荐方案**：NLLB-200（Meta 的 No Language Left Behind）
  - 支持 200 种语言，覆盖面极广
  - 对主流语言对（英↔中、日↔中等）翻译质量良好
  - 可在本地 CPU/GPU 上运行
- **备选方案**：MarianMT（Helsinki-NLP）
  - 轻量级，适合特定语言对
  - 对简单语言对（如西→英）效果较好

#### 3.2 在线翻译 — 传统翻译 API
- 支持接入云端翻译 API
- 可选服务商：DeepL / Google Translate / 百度翻译 / 有道翻译等
- 用户需自行配置 API Key
- 翻译质量稳定，适合简短句子的快速翻译

#### 3.3 在线翻译 — 大模型翻译
- 支持接入大语言模型（LLM）进行翻译
- **最低要求：兼容 OpenAI API 格式**（`/v1/chat/completions`）
  - 原生支持：OpenAI（GPT-4o / GPT-4o-mini 等）
  - 兼容接入：任何提供 OpenAI 兼容 API 的服务，包括但不限于：
    - DeepSeek
    - 通义千问（阿里）
    - Moonshot（月之暗面）
    - Claude（通过兼容网关）
    - 本地部署的 Ollama / LM Studio 等
- 用户可自定义配置：
  - API 地址（Base URL）
  - API Key
  - 模型名称
  - 翻译提示词（System Prompt，可选，提供默认模板）
- 大模型翻译优势：语境理解更强，长句和复杂表达翻译更自然

#### 3.4 引擎切换
- 用户可在设置中自由选择翻译引擎：「离线」/「在线翻译 API」/「大模型翻译」
- 三种模式互斥，切换即时生效

#### 3.5 翻译显示
- 原文与译文可同时显示（双行字幕）
- 用户可选择仅显示原文 / 仅显示译文 / 双语显示

> **离线翻译效果分析**：NLLB-200 在主流语言对上的翻译质量已接近 Google Translate 水平，对于实时字幕场景（短句为主）完全够用。DeepL 在欧洲语言上仍有优势，但覆盖语言数量有限（约 30 种）。建议默认使用离线翻译，用户对质量有更高要求时可切换到在线服务。

---

### 4. 悬浮字幕窗口

#### 4.1 始终置顶
- 字幕窗口支持 **始终置顶（Always-on-Top）** 模式
- 可覆盖在其他应用窗口之上显示

#### 4.2 自定义外观
- **位置**：用户可拖拽调整窗口位置
- **大小**：用户可调整窗口宽度和高度
- **透明度**：支持调节窗口背景透明度（0%~100%）
- **字体**：可自定义字体、字号
- **颜色**：可自定义文字颜色、背景颜色
- **显示行数**：可设置同时显示的字幕行数

#### 4.3 窗口模式
- **悬浮模式**：小窗口始终置顶，仅显示字幕
- **主窗口模式**：完整应用界面，包含所有控制选项和历史字幕

### 5. 自动保存

#### 5.1 保存格式
- 保存为带时间戳的纯文本文件（`.txt`）
- 格式示例：
  ```
  [2026-03-09 14:30:15] Hello, welcome to the meeting.
  [2026-03-09 14:30:15] 你好，欢迎参加会议。
  [2026-03-09 14:30:22] Let's start with today's agenda.
  [2026-03-09 14:30:22] 让我们从今天的议程开始。
  ```

#### 5.2 保存路径
- 默认保存路径：用户文档目录下的 `LingSubtitle` 文件夹
- 保存路径可在设置中自定义配置
- 文件名自动生成，格式：`字幕_YYYY-MM-DD_HH-mm-ss.txt`

#### 5.3 历史记录
- 提供历史记录列表页面，可查看所有已保存的字幕文件
- 支持按日期筛选和搜索
- 仅支持**查看**（只读浏览），不支持回放
- 支持打开文件所在目录

  - API Key

#### 6.4 翻译设置

- 在线翻译 API 配置（服务商、API Key）
- 大模型翻译配置：
  - API 地址（Base URL）
  - API Key
  - 模型名称
  - 自定义翻译提示词（System Prompt）
- 目标翻译语言

#### 6.5 字幕显示设置
- 字体、字号、颜色
- 背景颜色与透明度
- 显示行数
- 显示模式（原文 / 译文 / 双语）

#### 6.6 保存设置
- 自动保存开关
- 保存路径配置

## 非功能性需求

### 性能要求
| 指标 | 目标 |
|------|------|
| 字幕延迟 | < 2 秒（从音频发出到字幕显示） |
| 长时间运行 | ≥ 8 小时无崩溃、无内存泄漏 |
| CPU 占用 | 离线识别时 < 30%（中等配置 PC） |
| 内存占用 | < 1 GB（含模型加载） |

### 稳定性要求
- 长时间运行（≥ 8 小时）不出现内存泄漏
- 音频流中断后可自动恢复
- 异常情况下已录制的字幕不丢失（定时写入文件）

### 兼容性
- Windows 10 1903+ / Windows 11
- Mac 支持后续迭代（Tauri 2 本身跨平台）
